Windows Memory Forensics Cheat Sheet

Posted on 2019-12-17 Views:

来自“http://cybersecqa.blogspot.com/2018/03/windows-memory-forensics-cheat-sheet.html”

Acquire Target OS Version from Memory Image

fls -o 2048 -rp image.dd | grep config/software$

  • note the inode value

    icat -o 2048 image.dd 58832 > /outdir/software

    reglookup -p ‘Microsoft/Windows NT/CurrentVersion’ /outdir/software | grep ProductName

Acquire OS Processor Architecture

fls -o 2048 -rp image.dd | grep config/system$

  • note the inode value

    icat -o 2048 image.dd 58832 > /outdir/software

    reglookup -p ‘ControlSet001/Control/Session Manager/Environment/PROCESSOR_ARCHITECTURE” /outdir/system’

Perform Pagefile Analysis - String Search

python page_brute-BETA.py -r search.yar -f /outdir/pagefile.sys

Perform Process Analysis - Identify Anolmalies

vol.py -f memory.dd –profile=Win7SP1x64 pslist -v

  • identify incorrect spelling
  • identify unusual process names

    vol.py -f memory.dd –profile=Win7SP1x64 pstree

  • detect incorrect parent-child relationships

Process analysis - incorrect file paths

vol.py -f memory.dd –profile=Win7SP1x64 dlllist -v |less

process analysis - recover command line arguments

vol.py -f memory.dd –profile=Win7SP1x64

process analysis - identify process environment variable anomalies

vol.py -f memory.dd –profile=Win7SP1x64 envars –silent

  • use this command to look for Attacks on environmental variables.
  • the silent flag supresses known variables

Process anaylsis - the heap

vol.py -f memory.dd –profile=Win7SP1x64 heaps -p [pid]

  • use the heaps plugin to identify process heaps that may store application or network data.

Process analysis - detect search order hijacking w/ head anaylsis

vol.py -f memory.dd –profile=Win7SP1x64

Process analysis - detect backdoors w/ handle analysis

vol.py -f memory.dd –profile=Win7SP1x64 handles

  • look for handles to sockets

Process analysis - dkom detection w/ pscan & psxview

vol.py -f memory.dd –profile=Win7SP1x64 psscan

  • scans for process tags

Detect unlinked processes (DKOM)

vol.py -f memory.dd –profile=Win7SP1x64 psxview –apply-rules

  • ‘–apply-rules’ reduces false positives
  • alternate process listings:
    PsActiveProcessHead linked list
    EPROCESS pool scanning
    ETHREAD pool scanning (there is a referene to the EPROCESS owner)
    PspCidTable
    csrss.exe handle table
    csrss.exe internal linked list

Process analysis - dkom detection w/ csrss_pslist

vol.py -f memory.dd –profile=Win7SP1x64 csrss_pslist –output-file=csrss_pslist.html

  • The Client/Server Runtime Subsystem duplicates handles to all processes.
  • we can look for handles that don’t map to visible processes.

Process analysis - dkom detection w/ thrdscan

vol.py -f memory.dd –profile=Win7SP1x64 thrdscan

  • all processes have atleast one thread
  • we can look for handles that don’t map to visible processes.

Process analysis - dkom detection w/ filescan

vol.py -f memory.dd –profile=Win7SP1x64 filescan

  • FILE_OBJECT structures map to processes
  • we can look for files that don’t map to visible processes.

Process analysis - DKOM Detection w/ ldrmodules

vol.py -f memory.dd –profile=Win7SP1x64 ldrmodules -p [pid] –output=html –output-file=ldrmodules.html

Process - SIDs & Lateral Movement

vol.py -f memory.dd –profile=Win7SP1x64 getsids -v -p [target_pid]

Detect lateral movement

Process analysis - Profile process behavior

rip.pl -r software -f software > software.out

less software.out *look for the ‘profilelist’ section

Process analysis - privilege escalation

vol.py -f memory.dd –profile=Win7SP1x64 privs -p [target_pid]

  • We should be concerned about the following privs:
    SeBackupPrivilege - grants read access to all files
    SeDebugPrivilege - grants read/write access to other processes
    SeLoadDriverPrivilege - grants permission to load/unload kernel drivers
    SeChangeNotifyPrivilege - grants permission to register a callback function
    SeShutdownPrivilege - grants permission to reboot system

Process analysis - handles

vol.py -f memory.dd –profile=Win7SP1x64 handles -p [target_pid]

  • look for interesting file and network handles.
  • display all process handle types

Process analysis - review handles for signs of registry Persistence

vol.py -f memory.dd –profile=Win7SP1x64 handles -p [target_pid] -t Key

  • look for persistence mechanisms

Process analysis - Mutants

vol.py -f memory.dd –profile=Win7SP1x64 handles -p [target_pid] -t Mutant

  • review mutants
  • research the mutant names

Process analysis - handles to network sockets

vol.py -f memory.dd –profile=Win7SP1x64 handles -p [target_pid] -t File

  • look for handles to /Device/Tcp and /Device/Ipc
  • Identify malware supporting files

Process analysis - handles to remote network shares

vol.py -f memory.dd –profile=Win7SP1x64 handles -p [target_pid] -t File | grep -i Mup

  • Identify remote network shares

    vol.py -f memory.dd –profile=Win7SP1x64 sysmlinks -p [target_pid]

  • Identify remote network shares
  • Identify the data/time when the share was mounted

Process anaylsis - DLL Injection

vol.py -f memory.dd –profile=Win7SP1x64 dlllist -p [pid]

  • Look for unusual DLLs being used by the process.

    vol.py -f memory.dd –profile=Win7SP1x64 dlldump

  • And now we dump the suspicious DLL(s) for further analysis

Process analysis - Code Injection

vol.py -f memory.dd –profile=Win7SP1x64 malfind

  • We want to look for API hook artifacts such as push/ret, mov/call and push/jmp.
  • We can use the apihooks plugin to identify which functions are hooked.

    vol.py -f memory.dd –profile=Win7SP1x64 vadinfo

  • Prints verbose VAD info which includes memory protection levels

    vol.py -f memory.dd –profile=Win7SP1x64 vadtree

  • Prints VAD addresses in a tree view.

    vol.py -f memory.img –profile=Win7SP1x64 volshell -p [pid]

    db ([base offset])

  • We can use these commands to view specific regions of process memory.

0x01 PAGE_NOACCESS = VAD 0x0
0x02 PAGE_READONLY = VAD 0x1
0x04 PAGE_READWRITE = VAD 0x4
0x08 PAGE_WRITECOPY = VAD 0x5
0x10 PAGE_EXECUTE = VAD 0x2
0x20 PAGE_EXECUTE_READ = VAD 0X3
0x40 PAGE_EXECUTE_READWRITE = VAD 0x6
0x80 PAGE_EXECUTE_WRITECOPY = VAD 0x7
0x100 PAGE_GUARD
0x200 PAGE_NOCACHE
0x400 PAGE_WRITECOMBINE

Process - Code Injection 2

vol.py -f w225017.mem –profile=Win7SP1x64 malfind –dump-dir=outdir –yara-rules=.rules.yara -output=html -output-file=malware.html

  • Here we use yara to identify IOCs within processes

Process - Hollowing Detection

  1. Analyze the VAD to identify the named file at the base address.
  2. Use the getsids plugin to review SIDs and compare them to known good processes of the same type.
  3. Use the handles plugin to compare open handles (e.g., file, registry, sockets) to known good process of the same type.

DLLs - Incorrect Command Arguments

vol.py -f w225017.mem –profile=Win7SP1x64 dlllist -v |grep ‘Command’ |less

DLLs - DLLs Excuted by Rundll32.exe?

vol.py -f w225017.mem –profile=Win7SP1x64 dlllist -v |grep ‘rundll32’ |less

DLLs - DLL Function Exports

vol.py -f w225017.mem –profile=Win7SP1x64 impscan

  • Here we examine the process IAT to identify DLLs and then display exported functions from those DLLs

KERNEL - Identify Kernel Modules

vol.py -f w225017.mem –profile=Win7SP1x64 modules

  • Here we list object files loaded into kernel space.

KERNEL - Identify Malicious Kernel Drivers

vol.py -f w225017.mem –profile=Win7SP1x64 modules

  • Here we list object files loaded into kernel space.
  • Thee newer drivers are towards the end of the list
  • Dump all drivers and scan them with AV

SVCHOST - Services Running as Svchost

rip.pl -r software -f software > software.out

less software.out *look for the ‘svchost’ section

STRINGS - Correlate a String in Memory Dump to Process

srch _ string ‘some string’ memory.img > strings.txt

STRINGS - String search w/ output to text file

vol.py -f memory.dd –profile=Win7SP1x64 strings -s strings.txt > strings-output.txt

  • The output will show the processes and virtual address for each string.

DLLs - Check for File Tampering

vol.py -f memory.dd –profile=Win7SP1x64 procexedump -p [pid] -D .

  • Use the plugin dlldump for DLLs and moddump for drivers.
  • Compare dumped file MD5 to a known good file.

Process Analysis - List & Extract Accessible Pages

vol.py -f memory.dd –profile=Win7SP1x64 memmap -p [pid]

Process Analysis - Lists all pages accessibility to a process

vol.py -f memory.dd –profile=Win7SP1x64 memdump -p [pid]

  • Extracts all pages accessible to a process

cat ntuser.out | grep [file extension] |less
